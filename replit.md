# Oromo Legacy Wall

A Django-based memorial platform for preserving and honoring the lives and stories of Oromo individuals. Community members can submit "Legacies" (stories and photos) which are reviewed by zone-based moderators before appearing publicly on the memorial wall.

## Tech Stack

- **Framework:** Django 6.0.1 (Python 3.12)
- **Database:** SQLite (local development), PostgreSQL-ready via psycopg
- **Frontend:** Django Templates, CSS
- **Image Handling:** Pillow
- **Static Files:** WhiteNoise
- **Production Server:** Gunicorn

## Project Structure

- `legacies/` — Core Django app (models, views, forms, templates)
- `oromolegacy/` — Django project config (settings, urls, wsgi)
- `static/` — Source static files (CSS, images)
- `staticfiles/` — Collected static files (generated by collectstatic)
- `legacy_photos/` — User-uploaded memorial photos (MEDIA_ROOT)
- `db.sqlite3` — SQLite database
- `requirements.txt` — Python dependencies
- `manage.py` — Django management utility

## Key Models

- `Zone` — Geographic/administrative zones for moderation
- `ZoneModerator` — Links Django users to zones they moderate
- `Legacy` — Memorial entries with status (pending/approved/rejected)

## Running Locally

```bash
pip install -r requirements.txt
python manage.py migrate
python manage.py runserver 0.0.0.0:5000
```

## Deployment

Configured for Replit autoscale deployment using Gunicorn:
```
gunicorn --bind=0.0.0.0:5000 --reuse-port oromolegacy.wsgi:application
```

## Settings Notes

- `ALLOWED_HOSTS` includes Replit domains (`.replit.dev`, `.repl.co`, `.replit.app`)
- `CSRF_TRUSTED_ORIGINS` configured for Replit proxy
- `MEDIA_ROOT` points to `legacy_photos/`
- `STATIC_ROOT` points to `staticfiles/`
- WhiteNoise serves static files in both dev and production
